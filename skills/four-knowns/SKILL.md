# 四个知道 · 技术调研

本 Skill 引导对目标技术领域进行"四个知道"结构化调研，输出覆盖横向四知道和纵向四知道的完整报告。

**触发关键词**: 技术调研、行业调研、竞品分析、技术趋势、技术选型、行业分析、四个知道

## 调研流程

```
调研进度:
- [ ] 明确调研主题与范围
- [ ] 一（1）社会发展2-3年行业的趋势
- [ ] 一（2）竞争对手的情况
- [ ] 一（3）政策监控方向
- [ ] 一（4）技术发展
- [ ] 二（1）主要的paper，理论上能达到的，学术前沿
- [ ] 二（2）工业水平，最牛掰的公司达到的工程架构，市场上流行的
- [ ] 二（3）牛掰的架构，主要的工程模块
- [ ] 二（4）我们技术在行业中的位置，可以达到的程度
- [ ] 综合分析与行动建议
- [ ] 保存 Markdown 报告（必须）
- [ ] 询问用户是否导出 PDF（可选）
```

## 前置：明确调研主题

开始前向用户确认：

1. **调研领域**: 具体技术方向（如 LLM、自动驾驶、量子计算）
2. **业务场景**: 团队当前的业务上下文
3. **调研深度**: 快速概览 / 标准调研 / 深度研究
4. **重点关注**: 八个维度中是否有特别侧重

### （3）政策监控方向

调研要点：
- 国内政策：国家级和地方级相关文件
- 国际政策：美/欧/日韩的监管动态

## 综合分析

整合八个维度产出结论：

- **SWOT 分析**: 内外部因素综合研判
- **机会窗口**: 当前时间点的战略机遇
- **风险提示**: 需警惕的潜在风险（⚠️ 标记）
- **行动建议**: 短期(0-3月) / 中期(3-12月) / 长期(1-2年) 优先行动项
- **资源需求**: 支撑建议所需的投入估算

## 报告输出

调研完成后按以下流程输出：

### 第一步：保存 Markdown 报告（必须）

完成综合分析后，**必须**将完整报告保存为 `.md` 文件。

**文件命名**: `{调研主题}-research-{YYYY}.md`（如 `cross-platform-research-2026.md`）

**章节格式要求**（兼容后续 PDF 目录提取）：

```markdown
# 报告标题

## 1. 横向四个知道
### 1.1 社会发展 2-3 年行业趋势
### 1.2 竞争对手的情况
### 1.3 政策监控方向
### 1.4 技术发展

## 2. 纵向四个知道
### 2.1 学术前沿与理论极限
### 2.2 工业水平与标杆架构
### 2.3 最佳架构与工程模块
### 2.4 自身定位与发展路径

## 3. 综合分析
### 3.1 SWOT 分析
### 3.2 行动建议
```

保存完成后告知用户文件路径，并展示报告核心结论摘要。

### 第二步：询问是否导出 PDF（可选）

Markdown 保存后，**主动询问用户**：

> "报告已保存为 Markdown。是否需要导出为 PDF？"

**如果用户选择导出 PDF**，按优先级尝试以下方式：

#### 方式二：fpdf2 内联生成（零系统依赖备选）

```bash
pip3 install fpdf2
```

用 Python 脚本从 Markdown 内容直接生成 PDF：
- 使用系统自带中文字体（macOS: Arial Unicode；Linux: Noto Sans CJK）
- 手动构建封面、表格、正文排版

#### 方式三：提示用户使用本地工具

如果以上方式均不可行，提示用户：
- **VS Code**: Markdown PDF 插件
- **Typora**: 文件 → 导出 → PDF
- **Pandoc**: `pandoc 报告.md -o 报告.pdf --pdf-engine=xelatex`
- **浏览器**: 打开 .md 预览后 Ctrl+P 打印为 PDF

### 判断逻辑

```
完成综合分析后:
  1. 保存 Markdown 文件（必须执行，不需询问）
  2. 向用户展示核心结论摘要
  3. 询问 → "报告已保存为 {文件名}。是否需要导出为 PDF？"

  如果用户选择导出 PDF:
    尝试方式一 → 失败则尝试方式二 → 仍失败则提示方式三
    成功后打开预览（macOS: open; Linux: xdg-open）

  如果用户不需要 PDF:
    流程结束
```

## 输出规范

1. **数据有出处**: 关键数据标注来源，区分事实与推断
2. **时效性标注**: 标明数据获取时间
3. **置信度标记**: 不确定的判断标注置信度（高/中/低）
4. **可操作性**: 每个结论关联具体行动建议
5. **表格呈现**: 对比类信息优先用表格
6. **重点突出**: 关键结论 **加粗**，不确定信息 ⚠️ 标记

## 调研深度

| 深度 | 适用场景 | 输出要求 |
|------|----------|----------|
| 快速概览 | 初步了解某领域 | 每维度 3-5 条要点 |
| 标准调研 | 技术选型决策 | 每维度详细分析 + 对比表 |
| 深度研究 | 战略规划 | 完整报告 + 论文清单 + 路线图 |
