# 四个知道 · 技术调研

对指定技术领域进行系统性"四个知道"调研，覆盖**横向四知道**（宏观扫描）和**纵向四知道**（技术深挖）共八个维度，最终输出结构化报告。

## 安装

```bash
npx skills add paulgung/ths-skills@four-knowns
```

或安装整个 ths-skills 仓库：

```bash
npx skills add paulgung/ths-skills
```

## 触发方式

对话中使用以下语句即可激活：

- `帮我调研一下 [技术领域] 的发展情况`
- `对 [领域] 做一份四个知道分析`
- `做一份技术调研 / 竞品分析 / 行业分析`
- `我们在做 [技术方向]，帮我做一份深度技术调研`

**触发关键词**: 技术调研、行业调研、竞品分析、技术趋势、技术选型、四个知道

## 调研框架

### 横向四知道（宏观环境扫描）

| 维度 | 核心问题 |
|------|----------|
| （1）社会发展 2-3 年行业趋势 | 市场规模、增长预测、投融资动态、拐点信号 |
| （2）竞争对手的情况 | 竞争格局、玩家画像、差异化对比、商业模式 |
| （3）政策监控方向 | 国内外政策法规、行业标准、合规红线、政策机遇 |
| （4）技术发展 | 技术路线图、关键突破、技术瓶颈、开源生态 |

### 纵向四知道（技术深度剖析）

| 维度 | 核心问题 |
|------|----------|
| （1）学术前沿 | 顶级论文、理论极限、前沿方向、关键研究组 |
| （2）工业水平 | 标杆企业 Top 5、性能基准、市场主流方案 |
| （3）最佳架构 | 最优架构设计、核心模块拆解、技术栈推荐 |
| （4）自身定位 | 行业分位评估、差距分析、发展路线图 |

## 调研深度

| 深度 | 适用场景 | 输出规模 |
|------|----------|----------|
| 快速概览 | 初步了解某领域 | 每维度 3-5 条要点 |
| 标准调研 | 技术选型决策 | 每维度详细分析 + 对比表 |
| 深度研究 | 战略规划 | 完整报告 + 论文清单 + 路线图 |

## 输出产物

1. **结构化 Markdown 报告**（必须输出，自动保存为 `{主题}-research-{年份}.md`）
   - 八维度逐一分析
   - 综合 SWOT 分析
   - 分阶段行动建议（0-3月 / 3-12月 / 1-2年）
2. **PDF 导出**（可选，完成后 Agent 会主动询问）

## 使用示例

```
用户: 帮我调研一下大语言模型推理优化的技术现状

Agent: 好的，请先确认以下信息：
1. 调研领域：大语言模型推理优化 ✓
2. 业务场景：[请描述你们当前的业务上下文]
3. 调研深度：快速概览 / 标准调研 / 深度研究？
4. 特别关注哪些维度？
```

```
用户: 对 RAG 技术做一份四个知道分析，我们是做企业知识库的，标准调研即可

Agent: [开始逐维度输出，最终保存为 rag-research-2026.md]
```
